Our Contributor License Agreements

Pivotal uses various licenses to distribute software and documentation. To accept contributions, we generally require individuals or the organizations they are working for complete and sign Contributor License Agreements (“CLA”) based on the Apache Software License 2.0 CLAs. In all cases, contributors retain full rights to their contributions outside of Pivotal, while at the same time providing Pivotal the right to distribute and build upon such contributions in the Pivotal projects. Our CLAs establish a few things:

  • Individuals (or their organizations) grant a copyright license for their contributions to Pivotal
  • Individuals (or their organizations) grant a patent license for their contributions to Pivotal
  • The contribution is the individual’s or organization’s original creation, and the contributors remain free to do whatever they want with their contributions outside of Pivotal
  • Individuals (or their organizations) are not expected to provide support for their contributions.

With CLAs, we can encourage broad participation from the community while minimizing risks to Pivotal and our end-users resulting from insufficiently-licensed contributions coming into Pivotal’s open source projects.

Obvious Fixes

In certain cases described below, Pivotal may accept very minor contributions without requiring a CLA. This is designed for contributions that are minor enough (e.g. spelling errors, grammar mistakes, typos, formatting mistakes, etc.) to not be considered by you to be intellectual property that is independently protectable (an “Obvious Fix”). The purpose of this exception is to lower the barrier for new contributors to make helpful and necessary contributions to Pivotal projects while retaining the integrity of the project and our community.

What qualifies as an Obvious Fix

An obvious fix is a pull request that does not contain creative expression. We rely on your judgment to determine what is “obvious”; if you’re not sure, please ask the project owners or email cla@pivotal.io We generally consider Obvious Fixes to be changes that do not introduce any new functionality or creativity, for example:

  • Spelling/grammar mistakes
  • Typos
  • Comments clean-up
  • White-space or formatting changes
  • Logging messages or debug output
  • Changes to ‘metadata’ files that do not impact existing external dependencies such as .gitignore
  • Build script errors
  • Example configuration files
  • Project structure changes like renaming a build directory or changing a constant
  • Re-ordering of objects, variables or methods within a source file
  • Moving source files from one directory to another without code changes
  • Refactoring a source file into multiple source files or consolidating multiple source files a single source file without changing functionality

On the contrary, here are some contributions that would require a signed CLA before pull requests can be submitted/accepted:

  • Anything that results in a change in functionality
  • A new feature
  • A translation or refactor
  • Extensive or creative comments, README/docs changes, etc.

How to submit an Obvious Fix

If your contribution is an Obvious Fix, please add “Obvious Fix” to the pull request description.
